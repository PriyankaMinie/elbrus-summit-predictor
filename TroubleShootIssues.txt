# Airflow DAG — Issues, Fixes & Code Explanation

---

## Issues Faced and How They Were Solved

---

### Issue 1: DAG Import Error — `provide_context` not supported
**Error:**
```
TypeError: Invalid arguments were passed to PythonOperator (task_id: fetch_openmeteo).
Invalid arguments were: **kwargs: {'provide_context': True}
```
**Why it happened:**
Airflow 3.x removed the `provide_context=True` argument. In older versions you had to explicitly pass this to get `**context` in your function. In Airflow 3.x, context is passed automatically — you just declare the parameter you need (like `ti`) directly in the function signature.

**Fix:**
Changed function signatures from `def fetch_openmeteo(**context)` to `def fetch_openmeteo(ti)` and removed `provide_context=True` from PythonOperator.

---

### Issue 2: DAG not showing in the UI
**Symptom:** `airflow dags list` showed the DAG but the UI showed "0 Dags" or "No Dags found."

**Why it happened:** Two reasons:
1. The DAG was deactivated in the database from a previous failed import.
2. The UI filter was set to "Active" only — deactivated DAGs were hidden.

**Fix:**
- Clicked **Paused** filter in the UI to reveal hidden DAGs.
- Ran `airflow dags reserialize` to force Airflow to re-read all DAG files.

---

### Issue 3: Wrong DAGs folder path
**Error:**
```
Bundle 'dags-folder' path does not exist: /mnt/e/Portfolio_Project/dags
```
**Why it happened:**
When Airflow was first initialized (`airflow db init`), the working directory was `/mnt/e/Portfolio_Project`. Airflow set the `dags_folder` to that location. We had been placing the DAG file in `~/airflow/dags/` which Airflow was not watching.

**Fix:**
Created the dags folder inside the project and copied the file there:
```bash
mkdir -p /mnt/e/Portfolio_Project/dags
cp ~/airflow/dags/openmeteo_pipeline.py /mnt/e/Portfolio_Project/dags/
```

**How to check which folder Airflow is watching:**
```bash
airflow config get-value core dags_folder
```
Always put your DAG files in this folder.

---

### Issue 4: DAG file was empty after saving via VS Code
**Symptom:** `cat ~/airflow/dags/openmeteo_pipeline.py` returned nothing.

**Why it happened:**
VS Code was saving to the Windows filesystem. The file path `~/airflow/dags/` lives inside WSL's Linux filesystem — VS Code opened it but didn't save back to the right location.

**Fix:**
Used a Python one-liner to write the file directly from WSL:
```bash
python3 -c "
with open('/path/to/file.py', 'w') as f:
    f.write(content)
"
```
Going forward — always use `code /mnt/e/Portfolio_Project/dags/openmeteo_pipeline.py` (the Windows path via `/mnt/e/`) so VS Code saves correctly.

### Issue 5: WSL could not connect to PostgreSQL on Windows
**Error:**
```
psycopg2.OperationalError: connection to server at "172.31.64.1", port 5432 failed: Connection timed out
```
**Why it happened:**
PostgreSQL was only accepting connections from `localhost` (127.0.0.1). WSL has a different IP address and was being blocked by both PostgreSQL config and Windows Firewall.

**Fix — three steps:**
1. Added WSL IP range to `pg_hba.conf`:
   ```
   host    all    all    172.31.64.0/20    scram-sha-256
   ```
2. Confirmed `listen_addresses = '*'` in `postgresql.conf`
3. Added Windows Firewall rule in PowerShell (as Administrator):
   ```powershell
   New-NetFirewallRule -DisplayName "PostgreSQL WSL" -Direction Inbound -Protocol TCP -LocalPort 5432 -Action Allow
   ```
4. Restarted PostgreSQL service via `services.msc`

### Diagnostic Commands — Quick Reference

| What to check | Command |
|---------------|---------|
| Which folder Airflow watches | `airflow config get-value core dags_folder` |
| List all DAGs Airflow knows about | `airflow dags list` |
| Force re-read all DAG files | `airflow dags reserialize` |
| Test a DAG without running it | `airflow dags test openmeteo_pipeline 2026-06-09` |
| Check if dag-processor is running | `ps aux \| grep dag-processor` |
| Test PostgreSQL connection from WSL | `python3 -c "import psycopg2; conn = psycopg2.connect(host='172.31.64.1', database='elbrus_weather', user='postgres', password='YOUR_PASSWORD'); print('OK'); conn.close()"` |
| Check file content | `cat /mnt/e/Portfolio_Project/dags/openmeteo_pipeline.py` |
